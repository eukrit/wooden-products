What shipped in 0.4.0

New — Complete WPC Profiles Catalog

  • Single-page Leka-branded catalog at /catalog/ covering every core WPC profile Leka sources — 16 SKUs across 4 categories and 2 engineered lines.
  • Decking (8): 2 Signature Co-Ex (140×23, 148×22) + 6 Heritage Solid WPC (140×25 2D/3D, 140×20 2D/3D, 145×21 6-hole, 148×23 3D).
  • Cladding (3): Signature Co-Ex 156×21 + Heritage 148×21 in 2D and 3D finishes.
  • Wide Wall Panels (3): Signature 219×26 full-back, Signature 219×26 half-covered (vented), Heritage 219×26.
  • Fence (1): Signature Co-Ex 161.5×20 fence board — linked back to the existing /wpc-fence/ configurator.
  • Per-product card shows: inline SVG technical cross-section scaled to real dimensions, 3-up spec panel, surface finish chips, and all 8 Leka colourways.
  • Category tab filter (All / Signature / Heritage) wired per section via vanilla JS.
  • Extended catalog summary: 12 tiles documenting the 250+ additional unpriced profiles available on MOQ request (columns, joists, beams, bench strips, DIY tiles, ASA line).

New — Leka SKU Scheme

  • Scheme: LK[P|H]-[TYPE]-[WIDTH]-[THICKNESS][-VARIANT]. P = Premium Co-Ex / H = Heritage. Types: DK (decking), CL (cladding), PN (panel), FN (fence), plus CM/BN/JS/BM/TL for extended catalog.
  • data/catalog/leka-sku-map.json — internal-only mapping of every Leka SKU to the vendor code and source PI. Documents every product, unpriced catalog counts, and the full 8-colour palette.
  • All customer-facing surfaces reference Leka SKUs only — no vendor names, no factory codes.

Changed — Salesheet landing page

  • website/salesheet/index.html — now surfaces both entry points: /catalog/ (broad profile library) and /wpc-fence/ (fence configurator flow).
  • website/salesheet/Dockerfile — COPY catalog static/catalog so Cloud Run serves the new route.

Earlier — 0.3.0 (WPC Fence Configurator + Gemini render)

New — Interactive WPC Fence Configurator

  • Dedicated page at /wpc-fence/configurator/ with real-time SVG side-elevation preview.
  • Controls: Series (Premium / Classic), Bay width (1.5 / 1.8 / 2.0 / 2.9 m), Height (1.5 / 2.0 / 2.5 / 3.0 m), Gap slider 0–15 cm (0 = privacy, 1–15 = slatted), Fence run input, 8-swatch colour picker, single/double gate steppers.
  • Sticky summary panel with live bay count, post count, total boards, and total length with gates added in (single × 1.2 m, double × 2.4 m).
  • Quote modal pre-fills the full spec and submits to the existing /api/quote Slack lead flow.

New — Gemini Scene Render

  • New endpoint POST /api/render-scene calls Gemini 2.5 Flash Image ("Nanobanana") via REST to render the configured fence into one of five scenes: residential · hospitality · hospital · school · resort.
  • SHA-256 spec cache (re-runs for the same spec hit memory instantly).
  • Per-IP rate limit: 20 s. Daily budget guard: 200 renders/day (configurable via RENDER_DAILY_BUDGET).
  • Rendered image is attached to the Slack lead when the user submits a quote after rendering.

Changed — Parent WPC Fence page

  • New hero CTA "Configure your fence" + matching CTA under the Configurations section.
  • Applications expanded 3 → 5 cards to mirror the configurator scenes: Residential / Hospitality / Hospital / School / Resort.
